## Overview

The `Config` system provides a flexible, rule-based configuration for the TurkeyDPI engine. It supports JSON and TOML formats, validation, and hot-reloading.

## Config

### Structure

```rust
pub struct Config {
    pub global: GlobalConfig,
    pub rules: Vec<Rule>,
    pub limits: Limits,
    pub transforms: TransformParams,
}
```

### Fields

<ParamField path="global" type="GlobalConfig" required>
  Global engine settings and feature flags
</ParamField>

<ParamField path="rules" type="Vec<Rule>" required>
  Packet matching rules with transform actions
</ParamField>

<ParamField path="limits" type="Limits" required>
  Resource limits and safety constraints
</ParamField>

<ParamField path="transforms" type="TransformParams" required>
  Parameters for all transform types
</ParamField>

### Methods

#### load_from_file

```rust
pub fn load_from_file(path: impl AsRef<Path>) -> Result<Self>
```

Loads configuration from a JSON or TOML file (detected by extension).

<ParamField path="path" type="impl AsRef<Path>" required>
  Path to configuration file (.json or .toml)
</ParamField>

<ResponseField name="Config" type="Result<Config>">
  Loaded and validated configuration
</ResponseField>

**Example:**

```rust
use turkey_dpi_engine::Config;

let config = Config::load_from_file("config.toml")?;
```

#### from_json

```rust
pub fn from_json(json: &str) -> Result<Self>
```

Parses configuration from a JSON string.

**Example:**

```rust
let json = r#"
{
  "global": {
    "enabled": true,
    "enable_fragmentation": true
  },
  "rules": [
    {
      "name": "https-bypass",
      "match_criteria": {
        "dst_ports": [443],
        "protocols": ["tcp"]
      },
      "transforms": ["fragment", "padding"]
    }
  ]
}
"#;

let config = Config::from_json(json)?;
```

#### from_toml

```rust
pub fn from_toml(toml_str: &str) -> Result<Self>
```

Parses configuration from a TOML string.

**Example:**

```rust
let toml = r#"
[global]
enabled = true
enable_fragmentation = true

[[rules]]
name = "https-bypass"
transforms = ["fragment", "padding"]

[rules.match_criteria]
dst_ports = [443]
protocols = ["tcp"]
"#;

let config = Config::from_toml(toml)?;
```

#### validate

```rust
pub fn validate(&self) -> Result<()>
```

Validates the configuration, checking limits, parameters, and rule consistency.

**Example:**

```rust
let mut config = Config::default();
config.limits.max_flows = 0; // Invalid

assert!(config.validate().is_err());
```

#### merge

```rust
pub fn merge(&mut self, other: Config)
```

Merges another configuration into this one, replacing rules and settings.

**Example:**

```rust
let mut base_config = Config::default();
let overlay = Config::load_from_file("overlay.toml")?;

base_config.merge(overlay);
```

## Rule

### Structure

```rust
pub struct Rule {
    pub name: String,
    pub enabled: bool,
    pub priority: i32,
    pub match_criteria: MatchCriteria,
    pub transforms: Vec<TransformType>,
    pub overrides: HashMap<String, serde_json::Value>,
}
```

### Fields

<ParamField path="name" type="String" required>
  Unique name for the rule
</ParamField>

<ParamField path="enabled" type="bool" default="true">
  Whether this rule is active
</ParamField>

<ParamField path="priority" type="i32" default="0">
  Rule priority (higher = evaluated first)
</ParamField>

<ParamField path="match_criteria" type="MatchCriteria" required>
  Conditions for matching packets
</ParamField>

<ParamField path="transforms" type="Vec<TransformType>" required>
  List of transforms to apply when matched
</ParamField>

<ParamField path="overrides" type="HashMap<String, Value>" default="{}">
  Rule-specific parameter overrides
</ParamField>

**Example:**

```rust
use turkey_dpi_engine::config::*;
use std::collections::HashMap;

let rule = Rule {
    name: "discord-bypass".to_string(),
    enabled: true,
    priority: 100,
    match_criteria: MatchCriteria {
        dst_ports: Some(vec![443]),
        protocols: Some(vec![Protocol::Tcp]),
        domains: Some(vec!["discord.com".to_string()]),
        ..Default::default()
    },
    transforms: vec![
        TransformType::Fragment,
        TransformType::Padding,
        TransformType::Jitter,
    ],
    overrides: HashMap::new(),
};

rule.validate()?;
```

## MatchCriteria

### Structure

```rust
pub struct MatchCriteria {
    pub dst_ip: Option<Vec<String>>,
    pub src_ip: Option<Vec<String>>,
    pub dst_ports: Option<Vec<u16>>,
    pub src_ports: Option<Vec<u16>>,
    pub protocols: Option<Vec<Protocol>>,
    pub domains: Option<Vec<String>>,
    pub process: Option<String>,
}
```

### Fields

<ParamField path="dst_ip" type="Option<Vec<String>>">
  Destination IP addresses or CIDR ranges (e.g., `["8.8.8.8", "1.1.1.0/24"]`)
</ParamField>

<ParamField path="src_ip" type="Option<Vec<String>>">
  Source IP addresses or CIDR ranges
</ParamField>

<ParamField path="dst_ports" type="Option<Vec<u16>>">
  Destination ports (e.g., `[443, 8443]`)
</ParamField>

<ParamField path="src_ports" type="Option<Vec<u16>>">
  Source ports
</ParamField>

<ParamField path="protocols" type="Option<Vec<Protocol>>">
  Protocols: `tcp`, `udp`, or `icmp`
</ParamField>

<ParamField path="domains" type="Option<Vec<String>>">
  Domain names to match (requires SNI/Host inspection)
</ParamField>

<ParamField path="process" type="Option<String>">
  Process name (platform-dependent)
</ParamField>

**All criteria are AND-ed together. Empty/None means "match all" for that field.**

**Example:**

```rust
// Match HTTPS traffic to Discord
let criteria = MatchCriteria {
    dst_ports: Some(vec![443]),
    protocols: Some(vec![Protocol::Tcp]),
    domains: Some(vec!["discord.com".to_string()]),
    ..Default::default()
};

// Match all DNS traffic
let dns_criteria = MatchCriteria {
    dst_ports: Some(vec![53]),
    protocols: Some(vec![Protocol::Udp]),
    ..Default::default()
};

// Catch-all rule
let catch_all = MatchCriteria::default();
assert!(catch_all.is_catch_all());
```

## TransformType

### Enum

```rust
pub enum TransformType {
    Fragment,
    Resegment,
    Padding,
    Jitter,
    HeaderNormalization,
    Decoy,
    Reorder,
}
```

<ParamField path="Fragment" type="variant">
  Split packets into smaller fragments
</ParamField>

<ParamField path="Resegment" type="variant">
  Re-segment TCP streams
</ParamField>

<ParamField path="Padding" type="variant">
  Add random padding to packets
</ParamField>

<ParamField path="Jitter" type="variant">
  Add timing delays between packets
</ParamField>

<ParamField path="HeaderNormalization" type="variant">
  Normalize packet headers (TTL, window size, etc.)
</ParamField>

<ParamField path="Decoy" type="variant">
  Send decoy packets with low TTL
</ParamField>

<ParamField path="Reorder" type="variant">
  Reorder packet transmission
</ParamField>

## Complete Example

### TOML Configuration

```toml
# config.toml

[global]
enabled = true
enable_fragmentation = true
enable_jitter = true
enable_padding = true
enable_header_normalization = false
log_level = "info"
json_logging = false

[[rules]]
name = "discord-bypass"
enabled = true
priority = 100
transforms = ["fragment", "padding", "jitter"]

[rules.match_criteria]
dst_ports = [443]
protocols = ["tcp"]
domains = ["discord.com", "discordapp.com"]

[[rules]]
name = "twitter-bypass"
enabled = true
priority = 90
transforms = ["fragment", "decoy"]

[rules.match_criteria]
dst_ip = ["104.244.42.0/24"]
dst_ports = [443]
protocols = ["tcp"]

[[rules]]
name = "catch-all"
enabled = true
priority = 0
transforms = ["padding"]

[rules.match_criteria]
# Empty = match all

[limits]
max_flows = 10000
max_queue_size = 1000
max_memory_mb = 128
max_jitter_ms = 500
flow_timeout_secs = 120
log_rate_limit = 100

[transforms.fragment]
min_size = 1
max_size = 40
randomize = true

[transforms.padding]
min_bytes = 0
max_bytes = 64

[transforms.jitter]
min_ms = 5
max_ms = 50

[transforms.header]
normalize_ttl = false
ttl_value = 64
normalize_window = false
randomize_ip_id = true

[transforms.decoy]
send_before = true
send_after = false
ttl = 1
probability = 0.3
```

### Loading and Using Config

```rust
use turkey_dpi_engine::{Config, Pipeline};
use std::sync::Arc;

// Load from file
let config = Config::load_from_file("config.toml")?;

// Validate before use
config.validate()?;

// Use with Pipeline
let stats = Arc::new(Stats::new());
let pipeline = Pipeline::new(config, stats)?;

// Later: hot-reload configuration
let new_config = Config::load_from_file("config.toml")?;
pipeline.reload_config(new_config)?;
```
